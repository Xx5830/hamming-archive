# HamArc

Консольный архиватор с помехоустойчивым кодированием на основе кодов Хэмминга. Хранит файлы в формате `.haf`, обнаруживает и исправляет одиночные битовые ошибки в данных и метаданных без внешних зависимостей.

## Быстрый старт

```bash
# Собрать
cmake --preset release && cmake --build --preset release

# Создать архив
hamarc --create --file=archive.haf file1.txt file2.bin

# Извлечь
hamarc --extract --file=archive.haf
```

## Требования

- CMake ≥ 3.20
- Ninja
- GCC ≥ 13 или Clang ≥ 17 (поддержка C++23)

## Сборка

```bash
# Релиз
cmake --preset release
cmake --build --preset release

# Отладка + тесты
cmake --preset debug
cmake --build --preset debug
ctest --preset debug

# С AddressSanitizer
cmake --preset asan
cmake --build --preset asan
ctest --preset asan
```

Через `just` (если установлен):

```bash
just build          # релиз
just test           # отладка + тесты
just asan           # тесты с санитайзером
```

## Использование

```
hamarc КОМАНДА --file=<архив> [ОПЦИИ] [ФАЙЛЫ...]
```

### Команды

| Флаг | Краткий | Описание |
|---|---|---|
| `--create` | `-c` | Создать новый архив |
| `--extract` | `-x` | Извлечь файлы (все или указанные) |
| `--append` | `-a` | Добавить файл в архив |
| `--delete` | `-d` | Удалить файл из архива |
| `--list` | `-l` | Показать содержимое архива |
| `--concatenate` | `-A` | Слить несколько архивов в один |
| `--file=<путь>` | `-f <путь>` | Путь к архиву (обязательный) |

### Настройка кодирования

Каждый флаг принимает два числа: количество бит данных на блок и количество копий для репликации.

```
--encoding-data <биты> <копии>       # данные файлов          (по умолч. 256 1)
--encoding-size-file <биты> <копии>  # поле размера файла     (по умолч. 8 1)
--encoding-size-name <биты> <копии>  # поле длины имени       (по умолч. 8 1)
--encoding-name <биты> <копии>       # имя файла в заголовке  (по умолч. 8 1)
--encoding-meta <биты> <копии>       # метаданные кодирования  (по умолч. 8 1)

# Короткие алиасы: -e-d, -e-sf, -e-sn, -e-nf, -e-m
```

Чем больше бит на блок — тем меньше накладные расходы, но крупнее минимальная исправляемая единица. Копии > 1 добавляют репликацию с голосованием по большинству поверх кода Хэмминга.

### Примеры

```bash
# Создать архив
hamarc --create --file=backup.haf report.pdf photo.jpg data.csv

# Просмотреть содержимое
hamarc --list --file=backup.haf

# Извлечь все файлы в текущую директорию
hamarc --extract --file=backup.haf

# Извлечь один файл
hamarc --extract --file=backup.haf report.pdf

# Добавить файл в существующий архив
hamarc --append --file=backup.haf new_doc.txt

# Удалить файл из архива
hamarc --delete --file=backup.haf old_doc.txt

# Слить два архива
hamarc --create --file=merged.haf --concatenate part1.haf part2.haf

# Усиленное кодирование: блоки по 16 бит, 3 копии каждого блока
hamarc --create --file=robust.haf --encoding-data 16 3 critical.bin
```

## Формат .haf

```
┌─────────────────────────────────────────┐
│  Конфигурация кодирования × 5 копий     │  160 байт
├─────────────────────────────────────────┤
│  Заголовок файла 1 (код Хэмминга)       │
│  Данные файла 1    (код Хэмминга)       │
├─────────────────────────────────────────┤
│  Заголовок файла 2 (код Хэмминга)       │
│  Данные файла 2    (код Хэмминга)       │
└─────────────────────────────────────────┘
```

Заголовок содержит размер файла, длину имени, имя и параметры кодирования. Каждое поле заголовка само закодировано кодом Хэмминга. Конфигурация хранится пятикратно - при открытии восстанавливается голосованием по большинству, что защищает от повреждений в начале файла.
